## Project
NyayaDraft — a fine-tuned Indian legal document drafting system. Claude acts as a teacher model to generate a synthetic training dataset, which fine-tunes Qwen 2.5 7B via QLoRA. 11 doc types. Sold at Rs.199/doc via a Next.js + Razorpay app. Open source (MIT).

## Generation rule (STRICT)
- The training dataset is generated by the agent's OWN inference (Claude Code / Max), NEVER the Anthropic API.
- Do NOT run generate.py's API path. Do NOT ask the user to set ANTHROPIC_API_KEY.
- Use variation.py (local) for instructions, draft documents directly in the [[[INSTRUCTION]]]/[[[DOCUMENT]]]/[[[END]]] bracket format, validate with the local parse_response_text + legal_rules checker.
- Always use Opus 4.8 for dataset generation (keeps the dataset stylistically consistent).

## Dataset
- Target: 1,000 documents, weighted per-type counts + 80/10/10 split, both defined in config.yaml.
- Output file: out/full/dataset.jsonl. Raw drafts: out/full/raw.
- This dataset is gitignored by policy — never commit generated documents.

- Data-bug fixes already in place: partnership firm names use M/s (no LLP/Ltd/Pvt); no future-dated facts (date fields chained with relative_to); affidavit name-variants must be variants of the SAME person.
- A hook references ${CLAUDE_PLUGIN_ROOT} incorrectly and another calls python3 (should be python on Windows) — both non-blocking but noisy.
