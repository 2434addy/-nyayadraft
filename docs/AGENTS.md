# docs (W5 — costs & claim audit)

## Purpose
Operational documentation: the cost of generating the synthetic training data
(in USD and ₹) and the lawyer-facing audit of every statutory claim encoded in
the rule files.

## Key Files
| File | Description |
|------|-------------|
| `COSTS.md` | Generation-cost tables (per-document / pilot / full run; Batches vs sync API). Derived from the single source of truth in `data-pipeline/config.yaml`; the pipeline cost guard uses the same arithmetic. |
| `CLAIM_AUDIT.md` | Human-readable list of every `VERIFY`-tagged statutory claim across the rule files, for review by a qualified advocate. |
| `claim_audit.json` | Machine-readable backing data for `CLAIM_AUDIT.md`, regenerated by `data-pipeline/split_packages.py`. |

## For AI Agents

### Working In This Directory
- `COSTS.md` figures are computed from `data-pipeline/config.yaml` — if you change
  counts/pricing there, regenerate the tables rather than hand-editing numbers.
- `CLAIM_AUDIT.md` / `claim_audit.json` are generated by
  `data-pipeline/split_packages.py` from the authored doc-type packages; edit the
  source packages/rules, not these outputs directly.

### Testing Requirements
- None (documentation). Verify numbers trace back to `config.yaml` and the rule
  files after regeneration.

## Dependencies

### Internal
- `../data-pipeline/config.yaml` (cost inputs); `../legal_rules/rules/` (claim sources).
